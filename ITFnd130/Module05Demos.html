Overview

In this demo you will learn how to combine data from multiple tables and how to use intermediate queries to answer more complex questions. These are some of the most important query patterns in SQL because real database work usually involves related tables rather than just one table at a time.

A Simple Strategy for Building Joins

When students first learn joins, the syntax can feel harder than it really is. A good method is to build the query in four steps:

  1. List the columns you want in the result.
  2. List the tables that contain those columns.
  3. Figure out how the tables are connected.
  4. Write the join using those pieces.

Example: If you want publisher names and title names, then you need the Publishers and Titles tables, and they are connected by pub_id.

Select
  Pub_Name,
  Title
From dbo.Publishers
Inner Join dbo.Titles
  On Titles.Pub_Id = Publishers.Pub_Id;

1) Inner Joins

An inner join returns only the rows that have matching values in both tables. This is the most common type of join.

Look at the source data

Use Pubs;
Go

Select *
From dbo.Titles;

Select *
From dbo.Sales;
Go

Focus on the columns you need

Select
  Title_Id,
  Title
From dbo.Titles
Order By Title_Id;

Select
  Title_Id,
  Ord_Date,
  Qty
From dbo.Sales
Order By Title_Id;
Go

Join the two tables

Select
  Titles.Title_Id,
  Title,
  Ord_Date,
  Qty
From dbo.Titles
Inner Join dbo.Sales
  On Titles.Title_Id = Sales.Title_Id;
Go

Join more than two tables

Select
  Stores.Stor_Id,
  Stor_Name,
  Titles.Title_Id,
  Title,
  Ord_Date,
  Qty
From dbo.Titles
Inner Join dbo.Sales
  On Titles.Title_Id = Sales.Title_Id
Inner Join dbo.Stores
  On Sales.Stor_Id = Stores.Stor_Id;
Go

Use aliases for readability

Select
  [Store ID] = St.Stor_Id,
  [Store Name] = Stor_Name,
  [Title ID] = T.Title_Id,
  [Title] = Title,
  [Sales Date] = Ord_Date,
  [Sales Quantity] = Qty
From dbo.Titles As T
Inner Join dbo.Sales As S
  On T.Title_Id = S.Title_Id
Inner Join dbo.Stores As St
  On S.Stor_Id = St.Stor_Id;
Go

Try this: Run the inner join example and compare the results with the original Titles and Sales tables. Notice that only rows with matching Title_Id values appear in the result.

Joining Through a Bridge Table

Sometimes two tables are not directly connected. In that case, you often use a bridge table to connect them. In the Pubs database, Authors and Titles are connected through TitleAuthor.

Select
  [Author Name] = A.Au_Fname + ' ' + A.Au_Lname,
  [Title] = Title
From dbo.Authors As A
Inner Join dbo.TitleAuthor As TA
  On A.Au_Id = TA.Au_Id
Inner Join dbo.Titles As T
  On T.Title_Id = TA.Title_Id;
Go

2) Outer Joins

Outer joins are useful when you want to keep unmatched rows. For example, you may want all publishers listed, even if some of them do not currently have any titles.

Left Outer Join

Select
  Pub_Name,
  Title
From dbo.Publishers
Left Outer Join dbo.Titles
  On Titles.Pub_Id = Publishers.Pub_Id;

Right Outer Join

Select
  Pub_Name,
  Title
From dbo.Titles
Right Outer Join dbo.Publishers
  On Titles.Pub_Id = Publishers.Pub_Id;

Show only unmatched rows

Select
  Pub_Name,
  Title
From dbo.Titles
Right Outer Join dbo.Publishers
  On Titles.Pub_Id = Publishers.Pub_Id
Where Title Is Null;
Go

A common use of outer joins is to find missing related data. In this example, rows with Title Is Null show publishers that do not have matching titles.

Mixing Inner and Outer Joins

Be careful when mixing join types. An inner join later in the query can remove rows that were preserved by an outer join earlier in the query.

Select
  Pub_Name,
  Title,
  Qty
From dbo.Titles
Right Outer Join dbo.Publishers
  On Titles.Pub_Id = Publishers.Pub_Id
Left Join dbo.Sales
  On Titles.Title_Id = Sales.Title_Id;
Select
  Pub_Name,
  Title,
  Qty
From dbo.Titles
Right Outer Join dbo.Publishers
  On Titles.Pub_Id = Publishers.Pub_Id
Inner Join dbo.Sales
  On Titles.Title_Id = Sales.Title_Id;

Joining on Non-Foreign Key Columns

You can also join tables on matching values even when there is no formal foreign key relationship. This can be useful, but you need to think carefully about whether the match is truly meaningful.

Select
  Pub_Name,
  Au_Fname + ' ' + Au_Lname As Author_Name
From dbo.Publishers
Inner Join dbo.Authors
  On Publishers.City = Authors.City;

3) Cross Joins

A cross join returns every possible combination of rows between two tables. This can create very large result sets.

Select
  Pub_Name,
  Title
From dbo.Titles
Cross Join dbo.Publishers;
Go

Try this: Compare the row count from the cross join with the row count from the matching inner join. This helps show the difference between all possible combinations and only matching combinations.

4) Self Joins

A self join uses the same table twice in one query. This is useful when a table relates to itself, such as employees and their managers.

Use Northwind;
Go

Select
  Mgr.EmployeeId,
  Mgr.LastName,
  Emp.EmployeeId,
  Emp.LastName
From dbo.Employees As Emp
Inner Join dbo.Employees As Mgr
  On Emp.ReportsTo = Mgr.EmployeeId
Order By 1, 2, 3, 4;
Go
Select
  [Manager] = IIf(IsNull(Mgr.EmployeeId, 0) = 0, 'General Manager', Mgr.LastName),
  [Employee Name] = Emp.FirstName + ' ' + Emp.LastName
From dbo.Employees As Emp
Left Join dbo.Employees As Mgr
  On Emp.ReportsTo = Mgr.EmployeeId
Order By IsNull(Mgr.EmployeeId, 0), Emp.EmployeeId;
Go

Notice that the same table appears twice, but with different aliases. That lets SQL treat one instance as employees and the other as managers.

5) Unions

Joins combine columns from related rows. Unions combine rows from two result sets that have the same number and type of columns.

Use Pubs;
Go

Select
  Stor_Name,
  Stor_Address,
  City,
  State,
  Zip
From dbo.Stores

Union

Select
  Au_Lname + ' ' + Au_Fname,
  Address,
  City,
  State,
  Zip
From dbo.Authors
Order By 4, 3, 1;
Go

6) Subqueries

A subquery is a query inside another query. It is often used to calculate a value once and compare other results to it.

Grand total

Select
  [Grand Total] = Sum(Sales.Qty)
From dbo.Sales;

Compare state totals to the grand total

Select
  [State] = State,
  [Total By State] = Sum(Sales.Qty),
  [Grand Total] = (
    Select Sum(Qty)
    From dbo.Sales
  )
From dbo.Sales
Inner Join dbo.Stores
  On Stores.Stor_Id = Sales.Stor_Id
Group By State;
Go

Calculate percentages

Select
  [State] = State,
  [Total By State] = Sum(Sales.Qty),
  [Grand Total] = (
    Select Sum(Qty)
    From dbo.Sales
  ),
  [Percentage By State] = Str((100 * Sum(Sales.Qty)) / (
    Select Sum(Qty)
    From dbo.Sales
  ), 5, 2) + '%'
From dbo.Sales
Inner Join dbo.Stores
  On Stores.Stor_Id = Sales.Stor_Id
Group By State;
Go

7) Temporary Tables

Temporary tables let you store intermediate results and then reuse them in later queries. They are helpful when a complex report is easier to build in steps.

Select
  [State] = State,
  [Total By State] = Sum(Sales.Qty),
  [Grand Total] = (
    Select Sum(Sales.Qty)
    From dbo.Sales
  ),
  [Percentage By State] = Str((100 * Sum(Sales.Qty)) / (
    Select Sum(Sales.Qty)
    From dbo.Sales
  ), 5, 2) + '%'
Into #SalesByState
From dbo.Sales
Inner Join dbo.Stores
  On Stores.Stor_Id = Sales.Stor_Id
Group By State;
If Object_Id('tempdb..#SalesByState') Is Not Null
  Drop Table #SalesByState;

8) Common Table Expressions (CTEs)

A CTE is similar to a temporary result set, but it exists only for the statement that follows it. CTEs are often easier to read than temp tables for one-time reporting logic.

With SalesByState As (
  Select
    State,
    Sum(Qty) As TotalQty
  From dbo.Sales
  Inner Join dbo.Stores
    On Stores.Stor_Id = Sales.Stor_Id
  Group By State
)
Select *
From SalesByState;
Go

9) Variables

You can also store scalar values in variables and then reuse those values later in a query.

Declare @GrandTotal int;

Select
  @GrandTotal = Sum(Qty)
From dbo.Sales;

Select
  State,
  Sum(Qty) As TotalQty,
  @GrandTotal As GrandTotal
From dbo.Sales
Inner Join dbo.Stores
  On Stores.Stor_Id = Sales.Stor_Id
Group By State;
Go

Summary

In this demo you worked with several ways to combine and reshape data. Inner joins return only matching rows, outer joins help you keep unmatched rows, cross joins produce every combination, and self joins let you compare rows within the same table. Unions combine result sets vertically, while subqueries, temp tables, CTEs, and variables help you solve more complex reporting problems in manageable steps.
