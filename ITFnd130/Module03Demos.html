Select Statements and Query Clauses

Important

The SQL code on this page is provided as a demo and reference. You cannot run the code directly from this web page. To test these examples, copy the code, paste it into your SQL editor, and run it there one batch at a time. Read the code and compare the results carefully so you can see what each clause changes in the final result set.

Overview

This module focuses on the Select statement, which is the main tool used to retrieve data from a relational database. As you work through these examples, notice that a query is usually built one clause at a time. You decide what columns you want, where the data should come from, how it should be filtered, how it should be grouped, and how the final results should be sorted.

By the end of this demo, you should be able to read and write queries that use common clauses such as Select, From, Where, Order By, Group By, Having, Distinct, Union, and Top.

Using the Select Clause

The Select clause lists the columns or expressions you want in the result set. A query does not have to return only table columns. It can also return calculations, aliases, and literal values.

Select
  5 + 4,
  5 * 4;

Column aliases

Aliases rename columns in the result set. This is especially useful for calculations and reports.

Select
  [Sum] = 5 + 4,
  [Product] = 5 * 4;

In this course, square brackets are a good, reliable way to define aliases that contain spaces.

Use Northwind;
Go

Select
  [AVG Sales Price] = AVG(UnitPrice)
From [Order Details];

Select
  AVG(UnitPrice) As [AVG Sales Price]
From [Order Details];

Using the From Clause

The From clause tells SQL Server where the data should come from. That source can be one table, more than one table, a view, or even a derived result.

Select
  ProductName
From dbo.Products;

Select
  CompanyName
From dbo.Customers;

Select
  CompanyName
From Northwind.dbo.Customers;

You can also combine data from multiple tables.

Select
  Products.ProductName,
  Categories.CategoryName
From dbo.Products
Inner Join dbo.Categories
  On Products.CategoryId = Categories.CategoryId;

Table aliases

Aliases can also be used on table names to make longer queries easier to read.

Select
  P.ProductName,
  C.CategoryName
From dbo.Products As P
Inner Join dbo.Categories As C
  On P.CategoryId = C.CategoryId;

Using the Where Clause

The Where clause filters rows. If the condition is true, the row is returned. If the condition is false, it is not returned.

Use Northwind;
Go

Select
  ProductName,
  UnitsInStock
From dbo.Products
Where ProductName = 'Chai';

Using joins with filters

Select Distinct
  Customers.CompanyName,
  Orders.OrderDate
From dbo.Orders
Inner Join dbo.Customers
  On Orders.CustomerId = Customers.CustomerId
Where OrderDate = '8/25/97';

Wildcards with Like

Select
  ProductName,
  UnitsInStock
From dbo.Products
Where ProductName Like 'Ch%';

Select
  ProductName,
  UnitsInStock
From dbo.Products
Where ProductName Like 'C_a%';

Common operators

Select
  ProductName,
  UnitsInStock
From dbo.Products
Where UnitsInStock Between '0' And '35';

Select
  ProductName,
  UnitsInStock
From dbo.Products
Where UnitsInStock In ('0', '17');

Subquery example

Select
  CustomerId,
  CompanyName
From dbo.Customers
Where CustomerId Not In (
  Select CustomerId
  From dbo.Orders
);

Null checking

Select
  CompanyName,
  Fax
From dbo.Suppliers
Where Fax Is Null;

Use Is Null when checking for nulls. A comparison such as Fax = Null will not work the way students often expect.

Using the Order By Clause

The Order By clause sorts the result set. You can sort by one or more columns, and you can sort in ascending or descending order.

Select
  Pub_Id,
  Type,
  Title_Id,
  Price
From Pubs..Titles
Order By
  Pub_Id Desc,
  Type,
  Price;

You can also sort by column position, although naming the column is often clearer.

Using Aggregate Functions

Aggregate functions summarize data. Common examples include Sum, Avg, Min, Max, and Count.

Select
  [Grand Total] = Sum(ytd_sales),
  [Average Sales] = Avg(ytd_sales),
  [Number Of Sales] = Count(ytd_sales),
  [Number Of Entries] = Count(*)
From Pubs.dbo.Titles;

You can also use aggregate functions inside subqueries.

Select
  OrderId,
  CustomerId
From dbo.Orders
Where OrderDate = (
  Select Max(OrderDate)
  From dbo.Orders
);

Using Group By

The Group By clause is used with aggregate functions when you want summarized results for each group.

Select
  ProductName,
  Sum(Quantity) As TotalQty
From [Order Details]
Inner Join dbo.Products
  On [Order Details].ProductId = Products.ProductId
Group By ProductName
Order By ProductName;

Try this: Compare the grouped query with the non-grouped version that shows every order detail row. Notice how grouping reduces many detailed rows into one summary row per product.

Using Having with Group By

The Having clause filters groups after aggregation. In contrast, the Where clause filters rows before aggregation.

Select
  ProductName,
  Sum(Quantity)
From [Order Details]
Inner Join dbo.Products
  On [Order Details].ProductId = Products.ProductId
Where Quantity > 100
Group By ProductName
Having Sum(Quantity) > 200
Order By ProductName;

A helpful way to think about this is: Where filters rows, while Having filters groups.

Using Distinct

The Distinct keyword removes duplicate rows from the result set.

Select Distinct
  Orders.CustomerId,
  Orders.OrderDate
From dbo.Orders;

Keep in mind that duplication is evaluated across the entire row, not just one column.

Using Union

The Union operator combines rows from multiple queries into one result set. Each query must return the same number of columns in a compatible order.

Select
  [Customer Orders] = Count(*),
  [Year] = 'For 1996'
From dbo.Orders
Where Year(OrderDate) = 1996

Union

Select
  Count(*),
  'For 1997'
From dbo.Orders
Where Year(OrderDate) = 1997

Union

Select
  Count(*),
  'For 1998'
From dbo.Orders
Where Year(OrderDate) = 1998;
Go

Using Top

The Top clause limits the number of rows returned.

Use Northwind;
Go

Select Top (5)
  OrderId,
  ProductId,
  Quantity
From [Order Details]
Order By Quantity Desc;

With Ties includes additional rows that match the last ranked row.

Select Top (5) With Ties
  OrderId,
  ProductId,
  Quantity
From [Order Details]
Order By Quantity Desc;

Grouping for Subtotals

SQL Server also supports Rollup and Cube for subtotal and grand total reporting.

Select
  Title_Id,
  Sum(qty) As Quantity
From Pubs.dbo.Sales
Group By Title_Id With Rollup
Order By Title_Id;
Select
  Stor_Id,
  Title_Id,
  Sum(qty) As Quantity
From Pubs.dbo.Sales
Group By Stor_Id, Title_Id With Cube
Order By 1, 2;

Using Built-In Functions in a Select

You can also use built-in functions to improve how results are displayed.

Select
  IsNull(Stor_Name, 'All Stores'),
  IsNull(Title, 'All Titles'),
  Sum(qty) As Quantity
From Pubs.dbo.Sales
Inner Join Pubs.dbo.Titles
  On Sales.Title_Id = Titles.Title_Id
Inner Join Pubs.dbo.Stores
  On Sales.Stor_Id = Stores.Stor_Id
Group By Stor_Name, Title With Cube
Order By 1, 2;

Quick Summary of Common Clauses

Clause Main Job
Select Choose columns or expressions to return
From Choose the data source
Where Filter rows
Group By Create groups for aggregation
Having Filter groups after aggregation
Order By Sort the result set
Distinct Remove duplicate rows
Union Combine rows from multiple queries
Top Limit the number of rows returned
